# Notes App

A Flutter notes application built with Clean Architecture, Firebase, GetX, and GoRouter.

---

## Tech Stack

| Layer | Package |
|---|---|
| State Management | [GetX](https://pub.dev/packages/get) `^4.6.6` |
| Navigation | [GoRouter](https://pub.dev/packages/go_router) `^13.2.0` |
| Auth & Database | Firebase Auth + Cloud Firestore |
| UI | Google Fonts (Poppins), flutter_animate, iconsax |
| Utilities | uuid, intl |

## Architecture

This project follows **Clean Architecture** with three layers:

```
Presentation  →  Domain  ←  Data
```

- **Domain** layer is pure Dart — no Firebase, no Flutter imports. Defines entities, abstract repository contracts, and use cases.
- **Data** layer implements the contracts using Firebase. Models extend entities and handle Firestore serialization.
- **Presentation** layer contains GetX controllers and Flutter widgets. Controllers depend only on use cases, not on Firebase directly.

## Navigation (GoRouter)

| Route | Page |
|---|---|
| `/` | SplashPage |
| `/login` | LoginPage |
| `/register` | RegisterPage |
| `/home` | HomePage |
| `/add-note` | AddNotePage (pass `NoteEntity` via `extra` for edit mode) |

**Auth guard:** GoRouter's `redirect` callback checks `FirebaseAuth.instance.currentUser` on every navigation. Unauthenticated users are redirected to `/login`; authenticated users are redirected away from auth pages to `/home`.

**GetX + GoRouter integration:** `Get.key` (GetX's global navigator key) is passed as GoRouter's `navigatorKey`. This lets `Get.snackbar`, `Get.back()`, and `Get.context` work alongside GoRouter without needing `GetMaterialApp`.

## Real-time Notes

`HomeController` is registered in `DependencyInjection` with `fenix: true` so it is **never disposed** during navigation. Its Firestore stream (`StreamSubscription`) stays alive even while the user is on the Add Note page. When a note is written to Firestore, the stream emits the update immediately and the home list refreshes without any manual reload.

## Firestore Security Rules

Set these rules in **Firebase Console → Firestore → Rules**:

```js
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }

    match /notes/{noteId} {
      allow read: if request.auth != null && resource.data.userId == request.auth.uid;
      allow create: if request.auth != null && request.resource.data.userId == request.auth.uid;
      allow update, delete: if request.auth != null && resource.data.userId == request.auth.uid;
    }
  }
}
```

## Firebase Setup

1. Create a Firebase project at [console.firebase.google.com](https://console.firebase.google.com)
2. Enable **Email/Password** sign-in under Authentication → Sign-in method
3. Create a **Firestore Database**
4. Add your Android **SHA-1** fingerprint under Project Settings → Your apps
5. Run `flutterfire configure` to regenerate `firebase_options.dart`

To get your debug SHA-1:
```bash
cd android && ./gradlew signingReport
```
